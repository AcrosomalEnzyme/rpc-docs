title: txpool Namespace
---

`txpool` API 使您可以访问几种非标准的 RPC 方法，以检查包含所有当前挂起的交易以及排队等待未来处理的交易池的内容。

## txpool_content
可以查询“内容”检查属性以列出当前等待包含在下一个块中的所有交易的确切详细信息，以及计划仅用于未来执行的交易。

结果是一个具有“pending”和“queued”两个字段的对象。 这些字段中的每一个都是关联数组，其中每个条目将一个源地址映射到一批预定的事务。 这些批次本身是将随机数与实际交易相关联的映射。

请注意，可能有多个交易与同一个帐户和随机数相关联。 如果用户广播多个具有不同 gas 限额（甚至完全不同的交易）的消息，就会发生这种情况。

## txpool_inspect
可以查询`inspect`检查属性以列出当前等待包含在下一个块中的所有交易的文本摘要，以及仅被安排为将来执行的交易。 这是一种专门为开发人员量身定制的方法，可以快速查看池中的交易并发现任何潜在问题。

结果是一个具有“pending”和“queued”两个字段的对象。 这些字段中的每一个都是关联数组，其中每个条目将一个源地址映射到一批预定的事务。 这些批次本身是将随机数与交易摘要字符串相关联的映射。

### txpool_status
“状态”检查属性可以查询当前等待包含在下一个块中的交易数量，以及计划仅用于未来执行的交易数量。

结果是一个具有两个字段“待定”和“排队”的对象，每个字段都是一个计数器，表示该特定状态下的事务数。

### 代码示例

```javascript
> txpool.status
{
  pending: 10,
  queued: 7
}
```
